# App Checklist

Usar esta lista antes de considerar terminada una mini app.

## Producto

- El flujo principal esta resuelto.
- La primera pantalla ya es util.
- Hay datos de ejemplo o un estado vacio accionable.
- Los nombres de acciones son claros.
- Las acciones destructivas piden confirmacion.

## Datos

- Hay persistencia local si la app guarda estado.
- Hay import/export JSON si aplica.
- Hay validaciones de entrada.
- Hay manejo de errores visible.
- No se guardan secretos.

## UI

- Funciona en mobile y desktop.
- La navegacion es clara.
- Los controles no se superponen.
- Las tablas no rompen el layout.
- El contraste es suficiente.
- No hay texto largo apretado en botones.

## Codigo

- Un solo HTML completo.
- Sin archivos locales faltantes.
- Funciones ordenadas por responsabilidad.
- Estado, derivaciones, render y eventos son faciles de ubicar.
- Dependencias CDN minimas y utiles.

## Browser

- Abre con doble click.
- No requiere servidor.
- No falla si no hay datos previos.
- El reset vuelve a un estado razonable.
